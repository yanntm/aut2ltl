## `sweep_results.csv` — the 4248-language learner sweep

8496 rows = 4248 cases × 2 legs (`config_id ∈ {default, no-sat-exact}`); one row per
`(case_id, config_id)`. Header: `case_id,config_id,seed,ap_count,ref_classes,…,wall_seconds,verdict,detail`.

Reproduce the headline tally directly off this file:

```
python3 - <<'PY'
import csv; from collections import Counter
rows=list(csv.DictReader(open("reference/census/sweep_results.csv")))
keys=[(r['case_id'],r['config_id']) for r in rows]
print("rows", len(rows), "| distinct", len(set(keys)), "| dups", len(keys)-len(set(keys)))
print(sorted(Counter(r['verdict'] for r in rows).items()))
PY
```

## `e3_assembled.csv` — the ROLL FDFA baseline (E3)

16992 rows = 4248 cases × 4 kinds (`kind ∈ {periodic, syntactic, recurrent, ours}`);
header `case,kind,size,MQ,EQ`. The `ours` kind is **derived from the sweep** (our `N`
= the config-independent `ref_classes`; MQ/EQ = the default leg's
`n_member_total`/`n_equiv`), not recomputed — see `cluster_plan --e3` (commit
`0001e9526`) and `census_e3 --from-sweep` (commit `52aafffb1`). ROLL's three kinds are
the cluster run's FDFA sizes (all 3186/3186 ROLL modes OK; the `ours` timeouts on the
cluster were discarded in favour of the derivation).

