## `ablation_congruence.csv` — the fixpoint-congruence column (E2 / rows P9, P10)

The exact-ablation leg re-run with `fixpoint_congruent` recorded (spec §8 item 13):
6222 rows, one per language. The default leg is not re-run — its column is `true`
by construction — and E3 is untouched.

```
python3 -m tests.sosl.cluster_plan --legs ablate     # 1 case/command (the ceiling)
RUN=$(cluster/oarrun.sh --split 256 sosl/tests/sosl/logs/cluster/sweep.cmds)
python3 -m tests.sosl.congruence_column ../reference/census/ablation_congruence.csv
```

Graded (`ablation_congruence_summary.md`): **P9 clean** — all 3137 `ACCEPTOR_ONLY`
rows are `false`, as Theorem 5.3 requires; **P10 clean** — all 2336 `SOUND` rows
are `true`, so no byte-equality arises from a non-congruent partition; **dual
symmetric** over the 2733 comparable pairs; zero off-diagonal mass.

**The verdict partition is budget-censored — read the counts as a floor.** The
`ACCEPTOR_ONLY` / `SOUND` / `BUDGET` split depends on how much of the corpus a
given machine finishes inside the 60 s budget, so it is *not* reproducible across
drops: against `sweep_results.csv` this run moved 70 cases into `BUDGET` and 14
out of it. What is stable is the **classification**: no case ever flipped
`ACCEPTOR_ONLY ↔ SOUND` between the two drops. A permanence count taken from this
file is therefore a lower bound at a stated budget, not an invariant of the
corpus.
