# Drawing the Cayley graph of an invariant

The input is an `Invariant` `(C, key, λ, M, P, ε)`, and the object it presents is
`⟨𝒜, P⟩`. Only `𝒜` has a shape: what is *drawn* is the algebra — the classes and
the right action of the letters. `P` is **typeset, not drawn**: an accepting pair
is a pair of classes, not an edge, and forcing it into the graph as arrows is
what makes hand-drawn Cayley figures unreadable. So both backends carry it as a
one-line **caption** under the picture (`P = { ([b],[b]), … }`), which keeps the
object whole without lying about its type. The caption is ablatable (`--no-pairs`)
when only the bare algebra is wanted.

Three steps: name the letters, recompute the keys, classify.

## 1. Display letters fix an order

The machine alphabet orders letters by mask, i.e. by the characteristic tuple
with *absent < present*: for `AP = {a}` the order is `!a < a`. A figure wants
`a` before `b := !a`. A `Naming` therefore carries an explicit sequence of
`(letter, display_name)` — the caller's order — and it is that order, not the
machine one, that drives everything below. Letter *index* means index in this
sequence. With no renaming the sequence is the machine order, and the figure is
the machine's own view.

## 2. Keys are recomputed, never read

`inv.keys[c]` is the shortlex-least word of `c` **in the machine letter order**.
Under a renaming that reorders the letters, that word is no longer the least one,
so the figure recomputes:

> BFS from `[ε]`, popping FIFO, trying letters in display order; the first word
> that reaches a class *is* its key.

This is exactly the shortlex-least word in display order: BFS pops in
nondecreasing length, and within one length the pop order is the order the
parents were pushed, refined by letter index — which is lexicographic. `key(ε)`
is the empty word, displayed `ε`.

Two structural facts are asserted while walking, because a violation means the
file is not an invariant of an ω-language and every downstream picture is a lie:

- **Freshness.** No edge enters `[ε]`: the identity is *adjoined*, never the
  class of a non-empty word (`sosl/sos/algorithm.md`, the identity convention).
  So `[ε]` has in-degree 0 and the BFS root is a source.
- **Reachability.** Every class is reached. The algebra is generated by its
  letters, so an unreachable class is a malformed table.

## 3. The classification (what the drawing must show)

Nodes, one per class:

- **root** — `[ε]`. Drawn apart: it is a source, and it is the only class that is
  not the class of any word.
- **idempotent** — `M(c, c) = c`, identity excluded. These are the classes that
  can be the *loop* of a linked pair; the eye should find them without reading
  `P`.

Edges, one per `(class s, class c)`, from `s` to `M(s, c)`: **every column of the
multiplication table, not just the columns of the letter classes.** The
out-degree of every node is `|S|`, and what the picture shows is `M` itself.

Drawing only the columns `λ(x)`, `x ∈ Σ`, would show the *projection* of `M` onto
the letters — the transition system the letters generate. That is a strictly
weaker object: it determines `M` (the algebra is generated by its letters) but it
does not display it, and the figure is read as a table, not run as an automaton.
So all `|S|` columns are drawn, and the arrows that agree on a target fuse into
one, labeled with the comma-separated list of the classes that take it
(`[a],[b]`, `[a·b],[b·a]`) — the same fusion the letter columns already used,
now over the whole table.

- **key-tree edge.** For each non-identity class whose key is `w·x`: the edge out
  of `[w]` on the column `λ(x)`. Exactly `n-1` such edges, and they form a
  spanning tree rooted at `[ε]` — the BFS tree of step 2. They are not inked, but
  the layout leans on them (a key still reads as a path down from the root).
- **monochrome cycle.** Fix a column `c`; `s ↦ M(s, c)` is a *function* on
  classes, so its graph is a functional graph: every node has out-degree 1 and
  the reachable part of each component ends in exactly one cycle. Mark the edges
  on the cycles of length `≥ 2`. A **self-loop is a cycle of length 1 and is
  never marked** — the interesting structure is a class *rotating* through several
  classes under one column (a non-trivial group in the algebra, e.g. the `Z₂` of
  `Even`), and self-loops (of which there are many, every idempotent has one)
  would drown it.

  Found in one pass per column: follow `s ↦ M(s, c)` from each unvisited node,
  stamping a walk id; on meeting a node of the *current* walk, the segment from
  its first sighting to the present is a cycle. Linear, no strongly-connected-
  components machinery — out-degree 1 is the whole reason.

## 4. The identity is elided by default

`figure_of(inv, elide_identity=True)` — the default — drops `[ε]` from the
drawing: not as a node, and not as a label on any arrow. What it removes is
exactly what the identity axiom already fixes, and nothing else:

- **its column** `s·ε = s`, which contributes one `[ε]`-labeled self-loop at
  *every* node — `|S|` arrows that say only that `ε` is a right identity;
- **its row** `ε·c = c`, which contributes one arrow from `[ε]` to every class,
  each labeled with that class's own name — a copy of the class list.

Both are determined by the axiom, so a reader reconstructs them without looking,
and the ink they cost is `2·|S|` arrows on the densest part of the picture. With
them gone the drawing is the multiplication table of the semigroup `S(L)₊` (the
part with content) and the drawn out-degree is `|S| - 1`.

`elide_identity=False` keeps `[ε]` — the reading root, where a walkthrough of the
table starts, and the form in which the elision can be *shown* to cost nothing.
The two are meant to be drawn side by side once, and the elided one used from
then on.

**`𝒞`, in eliding figures only.** An arrow that carries *every* column is labeled
`𝒞` — the set of all non-identity classes — instead of listing them. It is the
longest label a full table produces and the least informative as a list, while as
a symbol it states the fact: a self-loop under `𝒞` **is** an absorbing class, and
a non-loop arrow under `𝒞` says every product lands there.

The collapse is off when the identity is drawn, and that is not a taste: `[ε]`'s
column is `s·ε = s`, so the `[ε]` entry of a row is a self-loop, *always*. A
self-loop therefore always carries `[ε]`, and an arrow that leaves its source never
can. One symbol for "every column" would then denote two different sets depending
on the shape of the arrow it sits on. The identity-keeping figure is the one that
has to be exact about the definitions, so it lists.

With the identity elided there is **no root node**, hence no init stub: the
backends draw the stub only when the figure has one. Keys are unaffected — they
are shortlex-least *words*, computed by the letter BFS of step 2 from `[ε]`,
which is present in the algebra whether or not it is drawn.

Finally, `Figure.pair_classes` / `Figure.pairs` give `P` as class pairs
`([s], [e])`, sorted shortlex by stem then loop — the caption line. Text, not
geometry. `P` never mentions `[ε]` (an accepting pair's loop is idempotent and
non-identity), so elision leaves the caption untouched. Note the caption is *not*
fed to the layout oracle: a graph label would shift `dot`'s bounding box, and only
the node coordinates are wanted from it.

## Layout

The model carries a **layer** per node (`len(key)`, the BFS depth) and nothing
else about geometry. The `dot` backend turns layers into ranks
(`rankdir=LR`, one `rank=same` group per layer), which places the root on the
left, the reachable-in-one-letter classes next, and the zero — always the
deepest — on the right. Any other backend is free to take the coordinates `dot`
computes (`dot -Tplain`) and re-emit them; that is the layout oracle, and it is
the caller's business, not this package's.
