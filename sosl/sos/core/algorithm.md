# core — constructing the syntactic ω-semigroup

This package computes, from a deterministic automaton, the **syntactic
ω-semigroup** of its language — the canonical algebra `S(L)₊¹` that Arnold's
congruence quotients out of `Σ*`, delivered as the invariant
`I(L) = (C, key, λ, M, P)` (`sos.invariant`). The paper of record is
[`research_notes/sos_constructed.md`](../../../research_notes/sos_constructed.md);
this document is its computational shadow, mapped onto the modules here. The
numbered claims below are the paper's (§ references); none is re-proved here.

**Input.** A deterministic, **complete**, transition-based Emerson–Lei
automaton `D = (Q, ι, δ, C, Acc)` over `Σ = 2^AP` — the form
`sos.build.importer.canonical` produces. No minimality is assumed anywhere:
the output is a function of the language alone, which is the point (§5).

**Output.** The canonical `Invariant`, or `None` when the monoid closure
blows its size cap — the cap is a parameter and blowing it is the caller's
policy (`sos.build` turns it into `ReferenceError`).

## The two obstructions, and the two keys

Two ω-regular languages are equal iff they agree on all **lassos** `u·v^ω`
(§2), so the algebra is machinery for sorting lassos into finitely many
types. Building the *syntactic* (coarsest, presentation-independent) such
algebra from an automaton hits two structural obstructions, each with its key:

1. **Recognition must see acceptance along runs** — a recognizer that keeps
   only endpoint states cannot evaluate a condition on infinitely-recurring
   marks. Key I: the acceptance-enriched monoid (`enriched.py`, §3).
2. **The syntactic congruence is two-sided** — Arnold quantifies over a left
   and a right context, but a monoid's closure table offers only right
   multiplication. Key II: the collapse + rotation lemma (`congruence.py`, §4).

## Key I — the acceptance-enriched monoid `EM(D)` (`enriched.py`, `closure.py`)

The enriched element of a finite word `w` is the map

```
    ⟦w⟧ : q ↦ ( δ(q, w),  mk(q, w) )
```

with `mk(q, w) ⊆ C` the marks on the run from `q` over `w`; composition is

```
    (f, M) · (g, N) : q ↦ ( g(f(q)),  M(q) ∪ N(f(q)) )
```

— a finite transformation monoid on `Q × 2^C`, generated by the letter
elements (`letter_elems`, one per alphabet mask). Facts carried from §3:

- **Skeleton lemma (3.2).** Two words with equal enriched elements induce
  identical run skeletons from every state — same states at block boundaries,
  same marks per block — hence equal acceptance inside every lasso context.
  So `⟦u⟧ = ⟦v⟧ ⟹ u ≈_L v`: `EM(D)` recognizes `L`, and the syntactic
  quotient factors through it (Corollary 3.3).
- **The enrichment is not optional (3.4).** A one-state automaton for `GF a`
  has a *trivial* transition monoid yet `a ≉_L b`: the marks-along-the-run
  are irreducible data. This is also why a group in the transition monoid
  proves nothing about the language — it can be pure encoding.

`closure.py` materializes `EM¹(D)` (identity adjoined as element 0) by BFS
from the letter elements under right-multiplication, up to the cap: the
element set, one shortest representative word per element (met
shortest-first in mask order, so representatives are shortlex-least), and
the right table `element × letter → element`.

## Key II — the congruence with right moves only (`congruence.py`)

Arnold's congruence (§2) declares `u ≈_L v` when interchangeable in both
context shapes — the two places a mutation can sit in a lasso:

```
    (linear)    ∀ x, y ∈ Σ*, t ∈ Σ⁺ :   x·u·y·t^ω ∈ L  ⟺  x·v·y·t^ω ∈ L
    (ω-power)   ∀ x, y ∈ Σ*         :   x·(u·y)^ω  ∈ L  ⟺  x·(v·y)^ω  ∈ L
```

By the skeleton lemma it transports to a relation `~` on `EM(D)`. Three
lemmas make `~` computable without ever multiplying on the left:

- **Collapse (4.1).** The acceptance of `w·z^ω` depends on the prefix only
  through the single state it reaches: `Acc(x, c) = A(st_x(ι), c)`, where
  `A(q, c)` iterates `c` from `q` — walk the functional graph of `st_c` to
  its closed cycle, union `mk_c` around the cycle (transient marks are
  finite), evaluate `Acc`. `profile(acc, elem)` computes the vector
  `Aprof(c) = (q ↦ A(q, c))` in one `O(|Q|)` pass.
- **Factorization (4.3).** `~ = ~lin ∧ ~ω`, with

  ```
      e ~lin f  ⟺  ∀ q ∈ Q   :  L(st_e(q)) = L(st_f(q))     (pointwise residual equality)
      e ~ω  f   ⟺  ∀ b ∈ EM¹ :  Aprof(e·b) = Aprof(f·b)     (right-invariant profile equality)
  ```

  The linear half is computed **once on `D`, with no monoid involved**:
  `residual_classes` delegates state language-equivalence to
  `spot.language_map` (deterministic input — one dualized complement per
  state), and an element's `~lin` datum is just its vector of reached-state
  classes. The ω half is a right-congruence condition seeded by profiles.
- **Rotation (4.4).** A left factor `a` acts on both seeds only by
  re-indexing the state slot — `st_{a·e}(q) = st_e(st_a(q))` for `~lin`, and
  `Aprof(a·e·b)(q) = Aprof(e·b·a)(st_a(q))` for `~ω` (conjugating the loop
  turns the left factor into a right extension). So the two-sided congruence
  is the coarsest **right**-invariant equivalence refining the single seed
  `(~lin class, Aprof)` — computable with the one operation the closure
  table gives for free.

`refine` realizes this: seed the partition by the pair, split a block
whenever right-multiplication by a letter separates two members, iterate to
fixpoint (Moore refinement, at most `|EM|` splits). **Theorem 4.5:** the
final blocks are exactly the classes of the syntactic congruence —
`EM(D)/~ = S(L)₊`.

Degenerate cases need no special-casing (§4): a prefix-independent language
has one residual class, `~lin` is total, and all discrimination rides on the
profiles (Proposition 4.6 — the `EvenBlocks` example); trivial acceptance
makes every profile constant and the quotient collapses onto the residual
algebra; the completion sink is an ordinary slot in every vector.

## The freeze (`quotient.py`) — word classes, fresh identity, accepting pairs

`pipeline` runs the stations above and hands `freeze` the closed monoid, the
class list, and the profiles. `freeze` produces the raw algebra and
canonicalizes it. Its one normative obligation is the **identity convention**
(§2, and `sos/algorithm.md`):

- **Word classes** are the `~`-classes of the elements reachable as images of
  **non-empty** words: every element the closure created past the identity,
  plus element 0 itself when it appears in the right table (some non-empty
  word then folds onto the identity element — e.g. `⟦!a⟧ = ⟦ε⟧` in the
  one-state `GF a`, or `⟦aa⟧ = ⟦ε⟧` in `EvenBlocks`).
- **`[ε]` is adjoined fresh, unconditionally** — a class no word can collide
  with, a two-sided unit on every class. Without this, a quotient over
  elements alone merges such a word into the identity and the class *count*
  becomes presentation-dependent, which breaks the byte-equality theorem
  (two `.sos` files byte-equal iff same language). A non-empty class may
  still act neutrally on all word classes (`[a·a]` in `Even`): it stays an
  ordinary class with a non-empty key; nothing to special-case.
- **Accepting linked pairs.** For word classes `s, e` with `e·e = e` and
  `s·e = s` (read off the class table), the verdict of `u·z^ω` with
  `[u] = s`, `[z] = e` is a pure table lookup:

  ```
      P(s, e)  =  Aprof(rep(e)) [ st_{rep(s)}(ι) ]
  ```

  — representative-independent because both seeds are constant on a class
  (`~lin` makes the reached states residual-equal, and `A(·, c)` is a
  membership question on those residuals). The fresh identity never enters a
  pair: an empty loop is not a lasso.

`canonical.py` then removes the last freedom — the class numbering: BFS over
`step(c, a) = M(c, λ(a))` from `[ε]`, letters in mask order, keys every class
by the first word reaching it (its shortlex-least word) and renumbers in
discovery order. This normal form is shared with the learner's export
(`sosl.learn.export`), which is what makes a reference and a learned
invariant byte-comparable. The membership read-off from `I(L)` — fold,
idempotent power, pair lookup — is specified in `sos/algorithm.md`.

## Cost

The dominant object is `|EM(D)|`, bounded by `(|Q|·2^{|C|})^{|Q|}`; the cap
converts the explosion into an honest `None` (the decision problems downstream
of this object are PSPACE-hard, so a cap somewhere is a necessity, not an
apology). Around it: residual classes cost one language-equivalence sweep on
`D`, once; profiles `O(|EM|·|Q|)`; refinement `O(|EM|·|Σ|)` per split; the
freeze is quadratic in the class count.

Canonical class counts `|S(L)₊¹|` that must not move under any change here
(guarded by `tests/sos/core_fingerprints.py`; the triptych rows are the
paper's Table 1, built from `research_notes/sos_figs/sources/`):

| language     | classes | note                                    |
|--------------|:-------:|-----------------------------------------|
| `GF a`       | 3       | the fresh-identity trap: `[ε], [a], [!a]` |
| `F a`        | 3       | —                                       |
| `a U b`      | 4       | —                                       |
| `GF(aa)`     | 6       | triptych; the presentation's `Z₂` dies in the quotient |
| `Even`       | 5       | triptych; decided on the stem           |
| `EvenBlocks` | 8       | triptych; prefix-independent, `~lin` blind |
| `F(a & Xa)`  | 6       | —                                       |

Canonicity is separately exercised by the two `GF(aa)` presentations
(`sos_figs/sources/gf_aa_parity.hoa` vs `gf_aa_reset.hoa` — a `Z₂`-carrying
and an aperiodic transition monoid, one identical invariant).

## Module map

```
              D  (deterministic complete Emerson–Lei form)
                   │
      ┌────────────┴────────────┐
 congruence.residual_classes   enriched.py ── letter elements (Key I)
 the ~lin base (spot)              │
      │                        closure.py ── EM¹(D), reps, right table
      │                            │
      │                        congruence.profile ── the ~ω seed
      └────────────┬───────────────┘
             congruence.refine ── the congruence ~ (Key II)
                   │
              quotient.py ── pipeline / freeze: word classes,
                   │         fresh identity, accepting pairs
              canonical.py ── the shared normal form -> Invariant
```

Layering: `core` consumes spot and the `sos` root data structures
(`Alphabet`, `Invariant`), nothing else; it never normalizes its input
(`sos.build.importer` owns that) and never owns resource policy (the cap is
a parameter). Consumers: `sos.build` (the reference builder and the
residual-automaton reader) and probes under `tests/sos/`. Ancestry: adapted
from the `aut2ltl/bls/definability` oracle pipeline, stripped of the
LTL-definability decision and certificate machinery — this package builds
the object; what one reads off it lives with its consumers.
