# genaut — exhaustive small-ω-automaton census

A standalone experiment (NOT wired into the `aut2ltl/` package): for a fixed
**shape**, exhaustively enumerate every tiny ω-automaton of that shape, reduce each
with Spot, and run `aut2ltl` over the survivors to measure coverage and surface
interesting cases. The generator is parametric over `Shape(n, k, c)` — `n` states,
`k` atomic propositions, `c` acceptance sets — and the bestiary spans every
tractable shape; see [`SHAPES.md`](SHAPES.md) for the map and the tractability wall.
Exhaustive enumeration does not scale far, so this is a one-off census per shape,
not a permanent corpus generator.

## The slot model

State `q0` is always initial. For every ordered pair `(src, dst)` and every
acceptance **markset** (subset of the `c` colours) there is one **edge slot**, whose
guard is any Boolean function over the `k` APs, with the all-false guard meaning
"edge absent". So `slots = n²·2^c`, each with `2^(2^k)` guard choices, giving
`N = (2^(2^k))^slots` raw automata. For `k = 1` the guard alphabet is `{0, a, !a, 1}`.

This slot set is **fully general** — a separate slot per markset covers parallel
coloured edges, and we deliberately do *not* hand-prune "useless" slots, so full
enumeration plus dedup cannot accidentally miss a language. The construction is
detailed in [`gen/algorithm.md`](gen/algorithm.md).

## Pipeline

1. **Generate** every combo for a shape (`gen/enumerate.py n,k,c`), running **one**
   `spot.postprocess(generic, high, small)` pass on each (keeps the acceptance
   family, no determinization; Small is a polynomial structural reducer).
2. **Dedup — two layers, in-memory and PRE-write** (a twin is never built into a
   file). First *byte-identical* (md5, first id wins); then *AP-canonical*: fold the
   `a ↔ !a` polarity / AP-rename twins via the shared `survey/normalize` key
   (`polarity ∘ names`). We dedup **only up to relabeling** — never by language or
   isomorphism — so genuinely differently-shaped encodings are KEPT (the round-trip
   recovering the same formula is what we measure, not assume).

   Each survivor is written `corpus/<tag>/<tag>_<id>.hoa` (self-identifying, id =
   generator position, reproducible via `gen/build.aut_at`), alongside a
   **`census.md`** recording combos / byte-distinct / polarity-fold / kept.

       python3 genaut/gen/enumerate.py 2,1,1     # -> genaut/raw/2state1ap1acc/
       # validate, then promote raw/<tag>/ -> corpus/<tag>/

3. **Survey** each shape with the repo harness — genaut's corpora are just more
   reference corpora, run and refreshed like those in
   [`../results/README.md`](../results/README.md) (rerun into scratch, diff,
   overwrite when clean), one committed run per shape under `logs/<tag>/`:

       python3 -m survey --folder genaut/corpus/2state1ap1acc \
           --logs genaut/logs/rerun/2state1ap1acc > .../SUMMARY.txt

4. **Analyse** a run into a frontier report (pure CSV, no tool re-run) — text
   digest + a multi-page PDF:

       python3 genaut/analyze_frontier.py genaut/logs/<tag>/default.csv \
           --out genaut/logs/<tag>/frontier.pdf

## Results

[`SHAPES.md`](SHAPES.md) is the bestiary map — every shape with its combos /
byte-distinct / polarity-fold / kept, **generated** from the `census.md` files by
`shapes_table.py`. Per-shape survey runs (the per-automaton CSV, SUCCESS summary,
frontier PDF) live under [`logs/<tag>/`](logs/). High-`kept` shapes are surveyed
separately.

## File map

    SHAPES.md        the bestiary map: every shape's combos/dedup/kept + the wall.
                     Generated by shapes_table.py — do not hand-edit the table.
    shapes_table.py  regenerate SHAPES.md's table from corpus/*/census.md.
    research_log.md  dated, dense log of observations (append-only).
    gen/             the generator (its own algorithm.md):
                       shape.py      Shape(n,k,c): slot model, guard alphabet,
                                     marksets, id<->combo bijection (pure).
                       build.py      a combo -> Spot TGBA (minterm-BDD guards) + reduce.
                       enumerate.py  driver: enumerate -> reduce -> 2 dedup gates ->
                                     write <tag>_<id>.hoa + census.md.
                                     `python3 genaut/gen/enumerate.py n,k,c [LIMIT]`
    analyze_frontier.py
                     one frontier report from a survey CSV (ventilation, idiom
                     collapse, size frontier, routes-to-true) — digest + PDF.
    probes/          diagnostics from the original 2state1ap1acc study (probe_post,
                     probe_true_collapse, probe_true_states); may reference the
                     legacy generator ids of that first census.
    raw/<tag>/       enumerate.py's scratch output (gitignored, regenerable). The
                     validated, git-tracked snapshot lives in corpus/<tag>/.
    corpus/<tag>/    the canonical automata, one subfolder per shape (git-tracked):
                     <tag>_<id>.hoa survivors + census.md. tag = "<n>state<k>ap<c>acc".
    logs/<tag>/      the committed reference run per shape: default.csv (per-automaton
                     survey), SUMMARY.txt (SUCCESS report), frontier.pdf.
