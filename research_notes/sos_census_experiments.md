# SoS Census — Experimentation Specification

**Status:** specification / declaration of intent. Everything below is to
be implemented; nothing below exists yet except where explicitly marked
*(exists in-repo)*. This document is the interface between the paper
(`sos_census.md`, [SωSN26]) and the implementation sessions: the paper's
⟨TBD⟩ tables cite the experiment ids below, and every number printed in
the paper is a *prediction or measurement* the tool must reproduce, per
the family discipline (`sos_learning_report.md` ledger style). Companion specs:
`sos_toltl_experiments.md` (strata testers, hunts), the symbolic spec
(derived-census driver at scale).

**One-line goal.** Provide the data for `sos_census.md`: the intrinsic
and derived censuses per `(N, m)` with their cross-validation, the
ventilated density sequences, the threshold theorems with minimal
witnesses, the cost-of-canonicity join, the hunts, the formula atlas —
and the standing regression harness that makes the census the family's
proving ground.

---

## 1. Objects consumed

- **The reference construction** producing `𝓘(L) = (𝒞, λ, M, P)` in
  `.sos` format from a deterministic EL automaton; byte-equality after
  canonical keying is the language-equality oracle *(exists)*.
- **The machine corpus** — the `genaut` census of small complete
  deterministic automata (2 states, 1 AP, small acceptance, …)
  *(exists)*; the derived census is this corpus pushed through the
  construction.
- **The triptych** `GF(aa)` (N = 6), `Even` (N = 5), `EvenBlocks` (N = 8)
  with hand-checked invariants [SωS26, Table 3] *(exist)* — E0's ground
  truth.
- **The read-off scans** of [SωS26, §7] (aperiodicity, safety–progress
  rung, acceptance index, prefix-independence) — wrap what the
  construction tool implements, itemize what is missing ⟨inventory at M1;
  the Wagner-degree read-off is expected missing and is its own work
  item, C7⟩.
- **The strata testers** of the extraction — condition (A) widths,
  condition (B) widths, layer statistics — are `sos_toltl_experiments.md`
  C1–C3; this spec *consumes* them as ventilation columns, never
  reimplements them.
- **The learner** [SωSL26] *(exists per project notes)* — cross-validation
  driver E3 runs it against census rows as membership oracles.

## 2. Components to implement (library level, no CLI polish needed)

**C1 — invariant validator.** Check a candidate tuple `(𝒞, λ, M, P)`
against the §3.2 axioms: associativity of `M`; fresh identity (`[ε]`
never a product of word classes); generation (every class reachable from
`[ε]` in the right Cayley graph of the letter classes); `P` a set of
linked pairs (`e² = e`, `s·e = s`, both non-identity). Pure table scans;
used by every other component as a gate.

**C2 — conjugacy saturation.** Compute the conjugacy classes of linked
pairs (`(s·x, y·x)` for `e = x·y`, when linked) and test/enforce that `P`
is a union of them. ⟨Theory gate: the paper's §3.2.2 conjugacy statement
awaits verification against [PP04] (library request). Implement the
stated closure now, and validate it empirically — see E4b, which tests
representation-independence of the membership rule directly; a
discrepancy is a paper edit, not a tool bug.⟩

**C3 — reducedness filter.** Decide whether the two-shape syntactic
congruence of the candidate's own language is the equality on `𝒞`:
the class-quantified context scans of [SωSX26, §4.2] — linear contexts
`(x, y, t)`, ω-power contexts `(x, y)`, verdicts through `Val` — merging
any two classes never separated. Non-reduced ⟹ discard (duplicate of a
smaller row). `O(N⁴)`-ish table ops; census sizes are tiny.

**C4 — canonical form and database.** Shortlex keying + `.sos`
serialization *(exists in the construction tool; wrap, don't rewrite)*;
both ventilation policies computed and stored — *labeled* (letters
distinguishable) and *up to letter renaming* (quotient by `Sym(m)` on
generators, orbit sizes recorded; which is primary is a paper decision
the data informs). The census database: rows keyed by canonical bytes,
indexed by `(N, m)` and every ventilation column, one flat file format
under `tests/**/logs/` conventions.

**C5 — intrinsic enumerator.** Generation-pruned, isomorph-rejecting
search: choose the `m` letter classes first, extend to an associative
table generated by them (fresh identity adjoined), reject non-canonical
isomorphs; per table, enumerate `P` over unions of conjugacy classes
(C2); filter by reducedness (C3); emit canonical rows (C4). The
semigroup-enumeration literature is the reference point ⟨library
request: small-semigroup enumeration, Distler et al.⟩ but the
generation constraint is ours and load-bearing — measure how hard it
prunes (E1's feasibility table).

**C6 — derived-census driver.** Batch the construction over the machine
corpus; canonical keying; deduplicate by bytes; record per row the
witnessing machines and the per-machine state count. Outputs: the
dedup ratio, the reach map (canonical sizes reachable at machine size
`n`), and the join table for E5. At corpus scale this is the symbolic
engine's first consumer — run explicit first, swap the engine in when
its spec's M2 lands.

**C7 — ventilation columns.** Per row, one scan each: aperiodicity;
ladder rung; acceptance index; Wagner degree ⟨implement; the read-off
is specified in [SωS26, §7] but not yet coded — its own work item⟩;
prefix-independence (residual count); (A)/(B) widths and layer stats
(import `sos_toltl_experiments.md` C1–C3); until-rank ⟨blocked on
[SωSX26] C6, which is blocked on the Thérien–Wilke source — the column
ships empty until then⟩.

**C8 — atlas builder.** Enumerate LTL formulas over `m` letters by
size (canonical syntactic order, dedup by trivial rewrites); per
formula, translate and run the construction (bounded per formula —
Spot translation under cap, skipped formulas recorded); stamp each
census row with its *first* hit — provably minimal for that row. Store
per LTL row the optimal formula, per non-LTL row the certificate of
[SωSX26, §4] (its extractor exists per that spec's E7). Each entry
conformance-checked offline: the stored formula's invariant must be
byte-equal to its row.

**C9 — calculus closure driver.** ⟨Gated on [SωSC26] operations being
implemented — Boolean combinations on aligned tables, rootings, inverse
substitutions, the AND-split (this last exists as [SωSX26] C7).⟩ Mark
each row *irreducible* if no operation over strictly smaller rows
produces it; fixpoint over the database, generate–reduce–lookup.

## 3. The conformance gates (mandatory, every experiment)

- **Idempotence.** Every emitted row, re-run through the reference
  construction on its own Cayley presentation, must return itself
  byte-exactly (reducedness + canonicity, checked by the tool that
  defines them). A mismatch is a stop-the-line bug in enumerator or
  construction — the two validate each other, which is the paper's §4.7
  point made operational.
- **Containment.** Derived ⊆ intrinsic at every size both cover; every
  intrinsic row at machine-reachable sizes is hit by some machine. A
  violation on either side is a stop-the-line bug (incomplete enumerator
  resp. buggy construction), never a statistic.
- Timeouts: per-instance cap 15s; a blown cap is `TIMEOUT`, recorded and
  reported — a finding, not an error. Long outputs to `tests/**/logs/`,
  one file per experiment id.

## 4. Experiments

### E0 — sanity on the worked examples (gates M1)

Run C1–C4 on the triptych's invariants. **Predictions to confirm:**
all three pass the axioms, conjugacy saturation, and reducedness; their
canonical bytes match the construction tool's output; the intrinsic
enumerator at `(N, m) = (5, 2)`, `(6, 2)`, `(8, 2)` finds `Even`,
`GF(aa)`, `EvenBlocks` among its rows. Any miss is stop-the-line.

### E1 — the intrinsic census and the partition function

Enumerate per `(N, m)`, `m = 2` first, `N` ascending to the wall.
**Paper deliverables:** the count sequence `#{L : |𝒞(L)| = N}` (both
ventilation policies; candidate OEIS submission), every C7 ventilation
as a sub-sequence, and the feasibility table (candidates generated /
surviving generation / surviving reducedness, runtime per `N`).
**Predictions (falsifiable guesses):** the aperiodic fraction dominates
at small `N`; prefix-independent rows are common; the residual stratum
((A) or (B) failing at every affordable width) is empty or near-empty
below the wall; the wall sits at single-digit `N` for `m = 2`.

### E2 — the derived census

C6 over the full machine corpus. **Paper deliverables:** the dedup
ratio (machines per language — the measured presentation redundancy),
the reach map in both directions (small machines with large algebras;
large machines wasted on small languages), and the derived row set for
E3/E5. **Prediction:** ratio well above 1 and growing with machine
size; the reach map shows canonical sizes both above and below machine
size on the same corpus.

### E3 — cross-validation, three ways

(i) Containment both directions (the §3 gate, run as an experiment with
a report, not just an assertion). (ii) **Learner loop:** for every row
(census-sized subset first), run the learner [SωSL26] against the row's
own membership oracle (`Val` on the table); the hypothesis must converge
to the byte-identical row. (iii) **Extraction loop:** for every LTL row,
run the extraction of [SωSX26]; its conformance gate re-derives the row
from the emitted formula. **Prediction:** all green; every red is a
minimal specimen attached to a stop-the-line bug in the named tool.

### E4 — threshold theorems

(a) For each property `X` (non-aperiodicity; each rung; each index;
each Wagner degree; each extraction stratum): the least `N` at which
`X` occurs over `m` letters, witness row attached. **Predictions to
confirm or refute** (either is a finding): `Even` is the minimal
non-LTL row (`N = 5`, `m = 2`, no non-LTL at `N ≤ 4`); `EvenBlocks` is
the minimal prefix-independent non-LTL row; `GF(aa)`'s `N = 6` is not
minimal for "LTL with a frozen layer needing (B) at width 2" ⟨low
confidence — that is the point⟩.
(b) **Conjugacy validation** (the C2 theory gate, empirically): for
every candidate `P` at census sizes, check that conjugacy saturation
coincides with representation-independence of the membership rule —
enumerate re-representations `u·v^ω = u′·v′^ω` up to a length bound and
compare verdicts. A discrepancy freezes §3.2.2's ⟨TBD⟩ the other way
and is a paper edit.

### E5 — the cost of canonicity

Join E1 and E2 per language: canonical size `|𝒞|` against the smallest
deterministic presentation in the corpus. **Paper deliverables:** the
ratio distribution, its tail, and the correlation of heavy cases with
ventilation classes — the paper's conjecture to state properly: blowup
concentrates on concatenation-shaped languages. **Prediction:** the
median ratio is near 1; the tail is real and classed.

### E6 — hunts (shared with [SωSX26] E6; the census is the instrument)

Standing scans over the database, re-run as the census grows:
- **H5** — a non-LTL row with all ω-power patterns constant
  (linear-only witness); exhaustion is evidence toward "F₂ always
  available".
- **H2/H3/H4** — smallest (A)-failure, smallest (B)-failure, smallest
  DG-fallback invocation (columns from C7; witnesses are paper figures).
- **H6** — smallest `k`-anchored layer (`k ≥ 2`) with a phase move under
  a neutral `k`-window (the width-`(k+1)` tightness witness of
  [SωSX26, §5.7]).
- **H7 (this paper's own)** — the smallest row *not* reachable from
  strictly smaller rows under the calculus (see E8): the first genuinely
  new language at each size.

### E7 — the formula atlas

C8 to the horizon the wall allows. **Paper deliverables:** fill rate
per size (all LTL rows at `N ≤ horizon` stamped); the succinctness map
(joint distribution of `|𝒞|` against minimal formula size — never
computed for this class); the offline conformance report; the handoff
artifact for [SωSX26] E4/E5 (its hit-rate and optimality-gap columns
calibrate against atlas entries). **Predictions:** `GF(aa)`'s row is
stamped by `GF(a ∧ Xa)` (up to the enumerator's canonical order) and
the triptych's non-LTL rows carry the byte-exact certificates of
[SωSX26] E7; most census rows are stamped by formulas of size well
below any extraction's output — the amputation bet's first data point.

### E8 — the basis *(gated on C9 / [SωSC26])*

Closure fixpoint over the database; mark irreducibles. **Paper
deliverables:** irreducible fraction per size (does the world thin out
into combinations?); correlation with subdirect irreducibility of
`(𝒞, M)` alone; the H7 witnesses. **Prediction:** the irreducible
fraction drops with `N` — the small ω-regular world is mostly
assembled.

### E9 — the proving ground (standing, not one-shot)

Package E0 + E3 + the §3 gates as the family's regression suite, run on
every construction/learner/extraction/calculus change, census subset
sized to the 15s cap per instance. Every failure ships with its minimal
census row. This is the paper's §4.7 as infrastructure; its report is
one line per run in the ledger.

## 5. Expected failures (read before filing bugs)

- **The enumeration wall is the object of study**, not an obstacle: when
  `(N, m)` blows past the cap, record the frontier (candidates/sec,
  where generation pruning saturates) — that is E1's feasibility table.
- **The conjugacy gate may move**: if E4b refutes the closure statement,
  C2/C5 outputs above the affected sizes are quarantined until the paper
  freezes the corrected clause.
- **Wagner column late**: C7's Wagner read-off is real work; ship the
  census with the column marked `PENDING` rather than hold E1.
- **Atlas enumeration is Spot-bounded**: skipped formulas (translation
  cap) are recorded; a row left unstamped because every small formula
  was skipped is `UNSTAMPED(cap)`, not evidence of minimality.
- **Learner divergence on a valid row** is stop-the-line for the
  learner, never triaged as census noise.

## 6. Milestones

- **M1** — C1–C4, E0 green; the database format frozen.
- **M2** — C6 derived census + E2, containment gate wired; C7 columns
  minus Wagner/until-rank; E3(i).
- **M3** — C5 intrinsic census + E1, E4 thresholds, E5 join; E3(ii–iii);
  E6 hunts first sweep.
- **M4** — C8 atlas + E7; Wagner column lands; E9 packaged as the
  standing suite.
- **M5** — C9 + E8, gated on the calculus implementation; H7.

Every milestone ends with a report appended to `sos_census_report.md`
(ledger style, one row per finding, predictions checked off or refuted —
a refuted prediction is a paper edit, not a footnote).
